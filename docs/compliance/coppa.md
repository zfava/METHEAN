# COPPA Compliance Posture

Draft for legal review. Last updated: 2026-04-17.

## Scope

METHEAN is a learning governance platform for homeschool families. It is operated by parents, not by schools. Parents create accounts, enroll their children, and control all aspects of the child's learning experience. Children interact with the platform through parent-supervised tutoring sessions and structured activities.

COPPA applies because the platform collects personal information from children under 13 through their educational activities (typed responses during tutoring, assessment answers, reading log entries).

## What Data We Collect from Children

All child data is collected with verifiable parental consent. Parents create the child's account and provide all identifying information. Children interact with the platform only through activities their parents have approved.

**Identifying information (provided by parent):**
- First name, last name
- Date of birth
- Grade level

**Educational activity data (generated by child during use):**
- Responses to tutoring prompts (typed text)
- Assessment answers
- Activity completion status and duration
- Reading log entries (book titles, pages, time spent)

**Derived data (computed by system, visible only to parent):**
- Mastery levels per learning node
- FSRS spaced repetition state
- Engagement patterns and pace metrics
- Wellbeing anomaly detections (parent-only)

See [data-inventory.md](data-inventory.md) for the complete field-level inventory.

## What We Do With Child Data

1. **Educational service delivery**: Tutoring, curriculum planning, mastery tracking, spaced repetition scheduling.
2. **Compliance reporting**: State-required hour tracking, subject coverage, attendance records, transcripts.
3. **Parent dashboards**: Progress visualization, governance review, wellbeing monitoring.
4. **AI-powered recommendations**: Curriculum suggestions, pacing advice, teaching guidance. All AI recommendations flow through parent governance rules before reaching the child.

## What We Do NOT Do

- We do not sell, rent, or share child data with any third party for marketing or advertising purposes.
- We do not use child data for behavioral advertising or targeted content outside the educational context.
- We do not display ads of any kind within the platform.
- We do not enable child-to-child communication or any social features.
- We do not create publicly accessible profiles for children.
- We do not retain child data after account deletion (see Deletion below).

## Third-Party Processors

Child-generated content (tutoring responses, assessment answers) is sent to AI providers for educational evaluation and tutoring. No child-identifying information (name, DOB) is included in AI prompts; only the content of the interaction and pseudonymous context (learning level, prior mastery state).

See [processors.md](processors.md) for the complete processor list and DPA status.

## Parental Rights

### Access (Inspection)

Parents can inspect all data collected about their children at any time through the platform's interface. A complete data export is available via `POST /api/v1/household/export`, which produces a ZIP file containing all household data in machine-readable JSON format.

**Verification**: `backend/app/services/data_export.py` and `backend/tests/test_data_export.py`

### Correction (Amendment)

Parents can edit their children's profiles, preferences, and learning data directly through the platform interface. Assessment records and portfolio entries can be updated or annotated.

### Deletion

Parents can request account deletion. Deletion cascades across all 49 household-scoped tables via PostgreSQL `ON DELETE CASCADE` foreign key constraints. This includes:

- All child identifying information
- All educational records (attempts, assessments, portfolios)
- All computed analytics (intelligence profiles, calibration data, style vectors)
- All wellbeing and family insight records
- All governance rules and events

Audit logs are anonymized (household_id removed) but the event record is retained for security compliance.

**Verification**: All models use `ForeignKey("households.id", ondelete="CASCADE")`. Verify with:
```bash
grep -c "ondelete.*CASCADE" backend/app/models/*.py
```

### Consent Withdrawal

Deletion of the account serves as consent withdrawal. No further data collection occurs after deletion. Data retained by third-party processors (AI providers) is subject to their respective retention policies; METHEAN does not have the ability to delete data already processed by Anthropic or OpenAI, but no identifying information is sent to those providers.

## Data Retention

- **Active accounts**: Data retained as long as the account exists.
- **Deleted accounts**: All household data hard-deleted via CASCADE. Anonymized audit logs retained.
- **AI provider data**: Governed by provider retention policies. Anthropic retains prompts for 30 days by default. OpenAI does not use API data for training.

## Security Measures

- **Row-Level Security**: PostgreSQL RLS policies on all 49 household-scoped tables prevent cross-household data access at the database level. See [rls-coverage.md](../rls-coverage.md).
- **Encryption in transit**: TLS 1.2+ required for all connections (HSTS enforced).
- **Authentication**: JWT tokens with dual-key rotation. CSRF double-submit cookie protection.
- **Access control**: Role-based (owner, co_parent, observer) with per-user permission overrides.
- **Wellbeing data isolation**: Wellbeing anomaly data is never returned by child-facing API endpoints.

## What Is Not Yet Implemented

The following COPPA-relevant capabilities are planned but not yet built:

- **Formal account deletion endpoint**: Deletion currently requires database-level operation. A self-service `DELETE /api/v1/household` endpoint is needed.
- **Parental consent verification flow**: The platform currently relies on the parent being the account creator (implied consent). A formal consent verification mechanism (email confirmation, knowledge-based authentication) is not yet implemented.
- **Data retention schedule automation**: Retention is currently "until deletion." No automated data aging or scheduled purging exists.

These items should be resolved before any school-district or enterprise deployment.
